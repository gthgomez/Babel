# Project Overlay — Example Autonomous Agent

## Purpose

Sanitized example overlay for an always-on autonomous agent that routes development work through Babel.

## Example Runtime Surface

- async task intake
- unattended execution
- governed tool usage
- persistent workspace memory

## Hard Invariants

- governance-first skills load before domain context
- unattended tasks must prefer explicit halt conditions over improvisation
- read and write authority must stay inside the declared workspace root
- operator-facing reports must be evidence-backed and deterministic

## Primary Objects

- workspace memory files
- agent configuration files
- Babel CLI runtime
- project workspaces under a single governed root
