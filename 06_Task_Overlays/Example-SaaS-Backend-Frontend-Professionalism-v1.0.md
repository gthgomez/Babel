# Example SaaS Backend Frontend Professionalism Overlay v1.0

## Purpose

Public-safe example delta for frontend professionalism work in an operational SaaS product.

Use this after the generic frontend professionalism overlay when polishing dashboard and settings surfaces.

## Product Tone

The product should feel:

- operational
- precise
- trustworthy
- evidence-oriented

The product should not feel:

- playful
- noisy
- over-marketed
- speculative

## Project-Specific Rules

1. Preserve auth and API boundaries.
2. Prefer status clarity over decorative emphasis.
3. Keep reports, logs, and settings readable before adding flourish.
4. Use compliance or status color only when the meaning is real.
